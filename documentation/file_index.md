## Scripts Directory

### scripts/init_TT01E_params.m
**Purpose**: Initialize all vehicle parameters  
**Type**: MATLAB script  
**Must run**: Before any simulation

**Contains**:
- Vehicle dimensions
- Mass and inertia properties
- Suspension parameters
- Drivetrain specifications
- Tire model parameters
- Simulation settings
- Input signal definitions

**Output**: 
- `params` structure in workspace
- `data/TT01E_params.mat` file

**Usage**:
```matlab
cd scripts
init_TT01E_params
```

### scripts/create_model.m
**Purpose**: Programmatically create Simulink model  
**Type**: MATLAB function  
**Run once**: On first setup or to rebuild model

**Contains**:
- Model creation code
- Block addition logic
- Basic connections
- Configuration setup

**Output**: 
- `models/TT01E_OpenLoop.slx` file

**Usage**:
```matlab
cd scripts
create_model
```

**Note**: Creates template; requires manual completion for full functionality

### scripts/run_simulation.m
**Purpose**: Execute simulation and plot results  
**Type**: MATLAB script  
**Primary tool**: For running simulations

**Contains**:
- Parameter loading
- Model configuration
- Simulation execution
- Result plotting
- Post-processing

**Requirements**:
- `init_TT01E_params.m` must be run first
- Model file must exist

**Usage**:
```matlab
cd scripts
run_simulation
```

**Output**:
- Simulation results in workspace
- Multiple plot figures
- Console output with status

### scripts/parameter_sweep.m
**Purpose**: Automated parameter variation study  
**Type**: MATLAB script  
**Advanced use**: Design optimization

**Contains**:
- Spring stiffness sweep example
- Gear ratio sweep example
- Mass variation study
- Automated result collection
- Comparative plotting

**Usage**:
```matlab
cd scripts
parameter_sweep
```

**Output**:
- Multiple analysis figures
- `data/parameter_sweep_results.mat`

**Customize**: Edit to study different parameters

## File Dependencies

### To run a basic simulation:

```
Required files (in order):
1. scripts/init_TT01E_params.m     → Creates parameters
2. scripts/create_model.m           → Creates model (first time)
3. models/TT01E_OpenLoop.slx       → The model
4. scripts/run_simulation.m         → Runs simulation

Generated files:
- data/TT01E_params.mat
- Workspace variables (simOut, params)
```

### Dependency diagram:

```
init_TT01E_params.m
        ↓
    params struct
        ↓
TT01E_params.mat ←─────┐
        ↓              │
run_simulation.m       │
        ↓              │
TT01E_OpenLoop.slx ────┘
        ↓
    Simulation
        ↓
Results (plots, data)
```

## Version Control Status

### Tracked files (committed to git):
- All documentation (*.md)
- All scripts (*.m)
- .gitignore
- LICENSE

### Ignored files (not committed):
- data/*.mat (generated)
- models/*.slx (generated, but could be tracked)
- *.asv (MATLAB autosave)
- slprj/ (Simulink cache)
- Any temporary files

### Should you commit models?

**Option 1**: Track .slx files
- ✅ Easier for non-technical users
- ✅ Version control of model structure
- ❌ Large binary files
- ❌ Difficult to merge changes

**Option 2**: Don't track .slx files (current approach)
- ✅ Small repository
- ✅ Easy merging
- ✅ Regeneratable from scripts
- ❌ Requires MATLAB to use

**Current approach**: Models are generated, allowing flexibility

## Editing Guidelines

### When editing parameters:
**Edit**: `scripts/init_TT01E_params.m`  
**Then run**: 
```matlab
init_TT01E_params
run_simulation
```

### When editing model structure:
**Edit**: `models/TT01E_OpenLoop.slx` in Simulink  
**Or edit**: `scripts/create_model.m` and regenerate

### When editing documentation:
**Edit**: Relevant `.md` file in `documentation/`  
**Preview**: Use GitHub or Markdown viewer

### When adding new features:
1. Add parameters to `init_TT01E_params.m`
2. Add model components to `create_model.m` or edit .slx
3. Update `run_simulation.m` for new outputs
4. Update documentation
5. Add examples if applicable

## File Naming Conventions

### MATLAB scripts:
- `init_*.m` - Initialization scripts
- `run_*.m` - Execution scripts
- `create_*.m` - Generation scripts
- `*_sweep.m` - Analysis scripts

### Documentation:
- `*.md` - Markdown format
- Lowercase with underscores for readability
- Descriptive names

### Models:
- `TT01E_*.slx` - Main models
- PascalCase for model names
- Descriptive suffixes (e.g., _OpenLoop, _ClosedLoop)
